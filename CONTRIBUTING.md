# Contributing to PYXIS3 projects

Thanks for considering a contribution. PYXIS3 is the open-source surface for [model-agnostic LLM serving infrastructure](https://pyxis3.ai). Contributions of code, documentation, recipes, benchmarks, or curated additions to the awesome list are all welcome.

## How to contribute

1. **Open an issue first** for substantive changes — design discussion lands better there than mid-PR
2. **Small fixes** (typos, docs, dead links, lint, dependency bumps) → PR straight in, no issue needed
3. **New recipes / benchmark targets / awesome-list entries** → PR with a one-line motivation in the description

## Standards

- **One concern per PR** — easier to review, easier to revert if needed
- **Tests + benchmarks if applicable** — PRs that change behaviour need to demonstrate the change
- **Reasonable commit messages** — `subject` + `body` if the change needs context
- **Markdown formatted** — most repos are markdown-heavy; respect the existing structure

## Code of conduct

Be excellent to each other. No identity-based attacks. Engage with the work, not the person.

## Licensing

Each repo declares its licence at the top level (`LICENSE` file). Contributions to a repo are accepted under that repo's licence — you retain copyright; we get a perpetual licence to redistribute under the repo's terms.
